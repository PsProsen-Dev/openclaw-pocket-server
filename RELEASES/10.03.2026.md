## 🎉 What's New

### 🦙 Local LLM Support (Experimental)

Run LLM models directly on your Android phone! This release introduces support for local inference engines:

#### node-llama-cpp Integration
- Prebuilt `@node-llama-cpp/linux-arm64` binary included
- Uses `--ignore-scripts` to skip 30+ minute compilation
- Works with GGUF format models (4-bit quantization recommended)

#### Ollama Support
- Full Ollama server integration
- One-command install: `curl -fsSL https://ollama.com/install.sh | sh`
- Model management via `ollama pull/run`

#### Model Recommendations
| Model | Size | RAM | Use Case |
|-------|------|-----|----------|
| TinyLlama 1.1B | ~670MB | 2GB | Testing |
| Phi-3 Mini | ~2.3GB | 4GB | Light tasks |
| Llama 3.2 1B | ~670MB | 2GB | Mobile-friendly |

<Warning>
  **Constraints**: Local LLM needs 2-4GB free RAM, 4GB+ storage. CPU-only inference is slow. Android has no GPU offloading for llama.cpp. For production, use cloud APIs.
</Warning>

## 🚀 Quick Start

### Install Local LLM

**Option 1: node-llama-cpp (Recommended)**
```bash
npm install -g node-llama-cpp --ignore-scripts
```

**Option 2: Ollama (Full Server)**
```bash
curl -fsSL https://ollama.com/install.sh | sh
ollama pull tinyllama
```

### Download a Test Model
```bash
mkdir -p ~/models
cd ~/models
curl -L -o tinyllama-1.1b-q4.gguf "https://huggingface.co/TheBloke/TinyLlama-1.1B-Chat-v1.0-GGUF/resolve/main/tinyllama-1.1b-chat-v1.0.Q4_K_M.gguf"
```

## 🔧 Technical Details

### Why `--ignore-scripts`?

The `node-llama-cpp` postinstall script attempts to compile `llama.cpp` from source via cmake:
- Takes **30+ minutes** on phone
- Often **fails** due to toolchain incompatibilities
- **Unnecessary** because prebuilt binaries work perfectly

Solution: Skip postinstall with `--ignore-scripts` and use the prebuilt `@node-llama-cpp/linux-arm64` binary.

### Architecture Changes

```
Android Device (Termux)
        │
        ▼
  glibc-runner (ld.so wrapper)
        │
        ▼
  Node.js v24 linux-arm64
        │
        ├──► OpenClaw Gateway (Cloud APIs)
        │    ┌─────┴──────┐
        │    │  AI CLIs   │
        │    │  SSH       │
        │    └────────────┘
        │
        └──► Local LLM Engine
             ├── node-llama-cpp (prebuilt)
             └── Ollama (optional)
```

## 📊 Comparison: Cloud vs Local

| Feature | Local LLM | Cloud LLM (OpenClaw) |
|---------|-----------|---------------------|
| Speed | Slow (CPU-only) | Fast (GPU-accelerated) |
| Privacy | ✅ Full privacy | Depends on provider |
| Cost | Free (after hardware) | Pay-per-token |
| Model Size | Limited by RAM | Unlimited |
| Setup | Manual | One command |
| Internet | Not needed | Required |

**Recommendation**: Use local LLM for testing/experimentation, cloud for production workloads.

## 📝 Migration Guide

### From OCA 09.03.2026

No breaking changes! Local LLM is an optional add-on.

**To enable Local LLM:**
```bash
# Update OCA first
oca --update

# Then install Local LLM support
npm install -g node-llama-cpp --ignore-scripts

# Or install Ollama
curl -fsSL https://ollama.com/install.sh | sh
```
